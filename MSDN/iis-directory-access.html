"тип входа в SQL Server учетка Windows" - приложения IIS обычно запускаются под своей учеткой, которая, скорее всего, не прописана на SQL сервере. Учетную запись можно изменить в конфиге IIS, в свойствах группы приложений (app pool). Или использовать вход  SQL Server.

"сетевой диск защищен паролем, в рабочей группе" - если защищен паролем, опять таки  учетка IIS в него скорее всего не войдет. нужно настроить права или поменять учетку

"Как определить пользователя под которым работает IIS?"

В Windows10 не знаю как. В Windows 2003 Server это делается так:

Панель управления - администрирование - Диспетчер служб IIS

Локальный компьютер - Группы приложений - DefaultAppPool - Свойства - Вкладка "Удостоверение"

"Как настроить учетку IIS, для подключения к сетевому диску?"

В свойствах общей папки, на вкладке безопасность, добавить нужного пользователя, с указанием имени компьютера (например MYCOMP\IISAPPPOOL). Поставить права на чтение и обзор содержимого.
